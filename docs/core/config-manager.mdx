`ConfigManager` stores per-project configurations that Aquila sends after the gRPC connection is established. It
extends [`BaseManager`](./base-manager.mdx) with key type `bigint` (projectId) and value type `ProjectConfiguration`.

The manager is updated automatically whenever Aquila sends new project configurations — you never write to it directly.

```ts
const config = action.configs.get(projectId);
config?.findConfig('API_KEY');
```

## Interfaces

### `ProjectConfiguration`

The configuration object stored for each project.

```ts
interface ProjectConfiguration {
  projectId: number | bigint;
  configValues: { identifier: string; value: PlainValue }[];
  findConfig: (identifier: string) => PlainValue | undefined;
}
```

### `ConfigurationDefinition`

Defines a per-project configuration field declared in the `Action` constructor. These are shown in the platform UI for
each project using the action.

```ts
interface ConfigurationDefinition {
  identifier: string;
  type: string;
  name?: Translation[];
  description?: Translation[];
  hidden?: boolean;
  optional?: boolean;
  linkedDataTypes?: string[];
  defaultValue?: PlainValue;
}
```
